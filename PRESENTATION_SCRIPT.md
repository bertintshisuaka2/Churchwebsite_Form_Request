## SLIDE 10: Trash System & Data Protection

**[Display: Trash tab screenshot]**

We've implemented a robust trash system to prevent accidental data loss:

**Soft Delete Process:**
1. Click delete button on any submission
2. Enter PIN (3495) to confirm
3. Submission moves to Trash tab (not permanently deleted)

**From Trash, administrators can:**
- **Restore**: Bring submission back to active list
- **Permanently Delete**: Remove forever (requires PIN + confirmation)

When permanently deleting, the system asks:
> "Are you sure you want to delete [Church Name]?"
- **Yes**: Permanently removes the submission
- **No**: Restores it to active submissions

This two-step process ensures critical data is never lost accidentally.

- HTTPS encryption
- JWT-based session management
- SQL injection prevention via ORM
